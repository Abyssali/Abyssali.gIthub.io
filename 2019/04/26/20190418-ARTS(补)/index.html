Rethinking Unit Test Assertions

写单元测试的时候，总感觉断言难写，是时候 重新思考单元测试断言 了

5 个每个单元测试必须回答的问题

  1. 被测的单元是什么？（模块？函数？类？或者是其他？）
  2. 它应该做什么？
  3. 实际输出是什么？
  4. 预期输出是什么？
  5. 你是怎样重现失败的？

简而言之： 简单的测试是更好的测试。
