每周一道算法题，leetcode

771. Jewels and Stones

没怎么做过leetcode，所以先从简单的题目开始了，这道题的思路就是将宝石字符串（Jewel）转为单个字符后存在易于查询的地方，再将石头字符串（Stone）转为 字符数组后遍历即可，在别人的分享中看到了比较有意思的答案，比如：一行代码完成解题：主要思路是使用正则表达式(正则原来还能这么用，学到了！）

每周阅读并点评一篇英文技术文档
What Is Readable Code?

这篇文章中，作者提到一个观点

我理解为代码本身也是需要输出自己(让别人读懂), 并指出了一种新的思路来判断代码可读性 - “Readable code should be easy to edit safely.” 并在之后讲述了自己认为的怎么可以使代码变得容易并安全的编辑。
并且作者在最后指出 简而言之，可读代码必须是可用的。也许，作为副作用，它看起来也很漂亮。
ps: 英文水平有点差，读起来还是挺吃力的，需要补补英文了！！！
